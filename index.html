mod_perl vs. PSGI benchmark

Introduction

Purpose of this research is compare performance of web applications based on mod_perl and PSGI. To take into account effects of multi-core systems, test were run on two machines. First one has two cores, and second four. In this research considered three parameters: number of handled request per second, the average time required to process single request and amount of RAM consumed.

1. Requests per second

Dependence of handled request per second and concurrency both mod_perl and PSGI shown in Fig. 1.1 and 1.2. With the growth of concurrency in both modes saturation reached quickly, however, PSGI mode allowed to handle approximately 20% more requests.

2. Request time

The average time required to process single request increases linearly with increasing concurrency both with mod_perl and PSGI. The difference becomes noticeable only in case of large value of concurrency. In PSGI mode request time was 15 - 20% less than in mod_perl.
